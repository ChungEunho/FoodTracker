# PRD Step 2 — 영양 정보 분석

## 목적
Step 1의 음식 인식 결과를 `openai/gpt-oss-120b:free` 모델에 전달하여 각 음식의 중량, 칼로리, 탄수화물·단백질·지방·당류 매크로 정보를 산출한다.

## 입력
| 항목 | 출처 | 형식 |
|------|------|------|
| 음식 구성 정보 | Step 1 출력 | `dict` (items 배열) |

## 처리 흐름

```
[Step 1 결과] → 프롬프트 조합 → GPT API 호출 → 영양 정보(JSON) → 포맷 출력
```

1. Step 1의 `items` 배열을 읽어 각 항목을 자연어 목록으로 변환한다.
2. `openai/gpt-oss-120b:free` 모델에 영양 분석 시스템 프롬프트와 함께 전송한다.
3. 모델 응답을 항목별 영양 정보 JSON으로 파싱한다.
4. 전체 합계(총 칼로리·매크로)를 계산하여 최종 결과에 포함한다.

## 시스템 프롬프트 방향
- 한국 음식 데이터베이스(식품안전처 기준) 기반으로 추정하도록 지시
- 1인분 기준 중량을 먼저 추정한 뒤 영양값을 계산하도록 순서 명시
- 불확실한 경우 최소·최대 범위를 함께 제공하도록 요청
- 출력 형식을 JSON으로 고정

## 출력 형식

```json
{
  "items": [
    {
      "name": "닭가슴살 샐러드",
      "weight_g": 200,
      "calories_kcal": 210,
      "carbs_g": 8,
      "protein_g": 35,
      "fat_g": 4,
      "sugar_g": 3
    },
    {
      "name": "현미밥",
      "weight_g": 210,
      "calories_kcal": 330,
      "carbs_g": 72,
      "protein_g": 6,
      "fat_g": 2,
      "sugar_g": 0
    }
  ],
  "total": {
    "weight_g": 410,
    "calories_kcal": 540,
    "carbs_g": 80,
    "protein_g": 41,
    "fat_g": 6,
    "sugar_g": 3
  }
}
```

## 에러 처리
| 상황 | 처리 방식 |
|------|-----------|
| Step 1 결과 없음 | 오류 메시지 출력 후 종료 |
| 모델 응답 파싱 실패 | 원문 응답 출력 후 합계 계산 생략 |
| API 오류 | 오류 코드와 메시지 출력 후 종료 |
