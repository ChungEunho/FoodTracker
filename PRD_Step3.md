# PRD Step 3 — 식사 히스토리 관리

## 목적
Step 2의 영양 분석 결과를 날짜·식사 종류와 함께 로컬 DB에 기록하고, 누적 섭취 현황을 조회·요약할 수 있는 히스토리 관리 기능을 제공한다.

## 데이터 모델

**테이블: `meals`**

| 컬럼 | 타입 | 설명 |
|------|------|------|
| `id` | INTEGER PK | 자동 증가 |
| `date` | TEXT | `YYYY-MM-DD` |
| `meal_type` | TEXT | `아침` / `점심` / `저녁` |
| `image_path` | TEXT | 원본 이미지 경로 |
| `items_json` | TEXT | Step 2 items 배열 (JSON) |
| `total_json` | TEXT | Step 2 total 객체 (JSON) |
| `created_at` | TEXT | 기록 시각 (ISO 8601) |

- 저장소: SQLite (`history.db`, 프로젝트 루트)
- 동일 날짜·식사 종류 중복 기록 허용 (덮어쓰지 않음, 수정은 삭제 후 재기록)

## 기능 목록

### 3-1. 식사 기록 저장
- 이미지 경로, 날짜, 식사 종류를 입력받아 Step 1 → Step 2를 순차 실행
- 결과를 `meals` 테이블에 INSERT

### 3-2. 일별 조회
- 특정 날짜의 아침·점심·저녁 기록과 각 식사별 총 칼로리·매크로를 출력
- 하루 합계(일일 총 섭취량)도 함께 표시

### 3-3. 기간별 요약
- 시작일~종료일 범위의 일별 칼로리 추이와 평균값을 출력

### 3-4. 기록 삭제
- `id` 또는 날짜+식사 종류 지정으로 단건 삭제

## CLI 인터페이스

```bash
# 식사 기록 (날짜 생략 시 오늘, 식사 종류 필수)
python step3_history.py log <이미지_경로> --meal <아침|점심|저녁> [--date YYYY-MM-DD]

# 일별 조회
python step3_history.py show [--date YYYY-MM-DD]   # 생략 시 오늘

# 기간별 요약
python step3_history.py summary --from YYYY-MM-DD --to YYYY-MM-DD

# 기록 삭제
python step3_history.py delete --id <ID>
```

## 에러 처리
| 상황 | 처리 방식 |
|------|-----------|
| 잘못된 날짜 형식 | 오류 메시지 + 올바른 형식 안내 |
| 잘못된 식사 종류 | `아침/점심/저녁` 중 하나 입력 안내 |
| 조회 결과 없음 | "해당 날짜의 기록이 없습니다" 출력 |
| DB 파일 없음 | 최초 실행 시 자동 생성 |
